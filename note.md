อาการ Error `listen tcp 0.0.0.0:5173: bind: An attempt was made to access a socket in a way forbidden by its access permissions` บนระบบ Windows ส่วนใหญ่เกิดจาก **พอร์ต (Port) 5173 ในเครื่องของคุณถูกใช้งานอยู่ หรือถูกระบบ Windows (Hyper-V/WinNAT) ล็อกเอาไว้ไม่ให้ใช้งานครับ**

วิธีแก้ไขมี 3 วิธีครับ (แนะนำให้ลองทำ **วิธีที่ 1** หรือ **วิธีที่ 2** ก่อน เพราะเจอบ่อยที่สุดใน Windows ครับ):

### 🌟 วิธีที่ 1: รีเซ็ตระบบจองพอร์ตของ Windows (แนะนำ ทำง่ายที่สุด)

บางครั้งระบบจำลองเครือข่ายของ Windows (WinNAT) จะเผลอล็อกพอร์ตบางช่วงเอาไว้ ทำให้ Docker ใช้งานไม่ได้ ให้ลองรีเซ็ตมันดูครับ:

1. กดปุ่ม **Start** พิมพ์ค้นหาคำว่า `PowerShell`
2. **คลิกขวา** ที่ Windows PowerShell แล้วเลือก **"Run as administrator"** (สำคัญมาก ต้องรันด้วยสิทธิ์แอดมิน)
3. พิมพ์คำสั่งนี้แล้วกด Enter:
```bash
net stop winnat

```


4. จากนั้นพิมพ์คำสั่งนี้แล้วกด Enter เพื่อเปิดใหม่:
```bash
net start winnat

```


5. กลับมาที่ VS Code หรือ Terminal ของคุณ แล้วลองสั่งรัน Docker อีกครั้งครับ:
```bash
docker-compose up -d

```



---

### 🌟 วิธีที่ 2: เปลี่ยนเลขพอร์ตในไฟล์ `docker-compose.yml` (ชัวร์ 100%)

ถ้าไม่อยากยุ่งกับระบบ Windows หรือพอร์ต 5173 ถูกโปรแกรมอื่นแย่งใช้ไปแล้วจริงๆ เราแค่เปลี่ยนให้ Frontend ไปใช้พอร์ตอื่น (เช่น 5174) แทนครับ:

1. เปิดไฟล์ `docker-compose.yml`
2. เลื่อนลงไปที่ service ชื่อ `frontend`
3. ตรงส่วน `ports:` ให้เปลี่ยนเลข **ตัวหน้า** เป็น `5174` (ตัวหลังปล่อยไว้เหมือนเดิม) แบบนี้ครับ:
```yaml
frontend:
  # ...
  ports:
    - "5174:5173"  # 🌟 เปลี่ยนจาก 5173 เป็น 5174

```


4. เซฟไฟล์ แล้วสั่งรันใหม่:
```bash
docker-compose up -d

```


*(หมายเหตุ: ถ้าใช้วิธีนี้ เวลาคุณเปิดหน้าเว็บ ต้องเข้าผ่าน `http://localhost:5174` แทนนะครับ)*

---

### 🌟 วิธีที่ 3: ปิดโปรแกรมที่แอบใช้พอร์ต 5173 ค้างอยู่

ถ้าคุณเคยสั่ง `npm run dev` รัน Vue.js ทิ้งไว้ข้างนอก Docker มันจะชนกันครับ

1. ใน PowerShell พิมพ์คำสั่งเพื่อหาว่าใครแอบใช้พอร์ตนี้อยู่:
```bash
netstat -ano | findstr :5173

```


2. ดูเลข PID ที่อยู่ขวาสุด (เช่น สมมติว่าเป็น `12345`)
3. สั่งบังคับปิดโปรแกรมนั้นด้วยคำสั่ง:
```bash
taskkill /PID 12345 /F

```


*(เปลี่ยน 12345 เป็นเลข PID ที่คุณเจอ)*
4. แล้วลองรัน `docker-compose up -d` ใหม่อีกครั้งครับ

ลองทำ **วิธีที่ 1** ดูก่อนนะครับ มักจะแก้ปัญหานี้บน Windows ได้ทันทีเลยครับ!
