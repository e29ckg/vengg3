# ระบบบริหารจัดการเวรนอกเวลาทำการ

ระบบแอปพลิเคชันบนเว็บ (Web Application) สำหรับจัดการตารางเวรนอกเวลาทำการของหน่วยงาน รองรับการจัดเวรแบบลากวาง (Drag & Drop) การตรวจสอบเงื่อนไขเวรซ้ำซ้อน ระบบขอแลกเปลี่ยนเวรระหว่างเจ้าหน้าที่ และการออกรายงานเป็นเอกสาร

## 🛠️ เทคโนโลยีที่ใช้งาน (Tech Stack)

* [cite_start]**Frontend:** Vue.js 3 ทำงานร่วมกับ Vite [cite: 560, 602, 676]
* [cite_start]**UI Framework:** Bootstrap 5 (พร้อม Popper.js สำหรับ Dropdown) [cite: 663, 672, 676]
* [cite_start]**Backend:** PHP (Native API / PDO) [cite: 560, 600, 810]
* [cite_start]**Database:** MySQL [cite: 560, 810]
* [cite_start]**Libraries เพิ่มเติม:** * `axios` สำหรับเชื่อมต่อ API [cite: 609, 612]
    * [cite_start]`sweetalert2` สำหรับแจ้งเตือน Pop-up [cite: 687]
    * [cite_start]`pizzip`, `docxtemplater`, `file-saver` สำหรับสร้างและดาวน์โหลดไฟล์ Word (.docx) [cite: 555, 556]
* [cite_start]**Infrastructure:** Docker และ Docker Compose [cite: 560, 810, 815]

## ✨ ฟีเจอร์หลัก (Key Features)

### 1. ระบบจัดการข้อมูลพื้นฐาน (Master Data & User Management)
* [cite_start]**การจัดการผู้ใช้งาน:** แอดมินสามารถเพิ่ม ลบ และแก้ไขข้อมูลเจ้าหน้าที่ พร้อมระบบเข้ารหัสรหัสผ่าน (Bcrypt) และใช้ UUID เป็นรหัสประจำตัวเพื่อป้องกันข้อมูลซ้ำซ้อน [cite: 605, 608, 640, 643]
* [cite_start]**ระบบสถานะและลำดับอาวุโส:** รองรับการตั้งค่าสถานะผู้ใช้งาน (เช่น ระงับการใช้งาน) ซึ่งจะผลักรายชื่อไปอยู่ท้ายตารางอัตโนมัติ และมีระบบ Soft Delete ที่เก็บข้อมูลประวัติไว้ [cite: 615, 893, 894, 895, 905]
* [cite_start]**การตั้งค่าหน่วยงาน:** จัดการข้อมูลชื่อหน่วยงาน โลโก้ และรายชื่อผู้บริหารที่จะแสดงในรายงาน [cite: 951, 955]
* [cite_start]**การจัดการหน้าที่เวร:** ตั้งค่าเวรหลัก หน้าที่ย่อย กำหนดค่าตอบแทน เลือกสีประจำป้ายเวร (Color Picker) และเรียงลำดับความสำคัญด้วยการลากวาง [cite: 686, 688, 693, 800, 802]

### 2. ระบบจัดตารางเวร (Shift Scheduling)
* [cite_start]**การสร้างคำสั่งเวร:** สามารถระบุเดือนและเลือกวันที่ที่จะให้มีเวรปฏิบัติงานจริงได้ [cite: 727, 778]
* [cite_start]**ปฏิทินจัดเวรแบบ Interactive:** แอดมินสามารถจัดเวรโดยการลากรายชื่อผู้มีสิทธิ์ไปวางในปฏิทิน (Drag & Drop) และสามารถลากย้ายสลับวันภายในปฏิทินได้ [cite: 756, 763, 797]
* [cite_start]**จัดเวรอัตโนมัติ (Auto Assign):** ระบบคำนวณและกระจายเวรให้ผู้มีสิทธิ์ตามลำดับ (Round Robin) และตามจำนวนคนที่ต้องการต่อวันได้อัตโนมัติ [cite: 792, 794, 939, 940]
* [cite_start]**ระบบตรวจสอบสมองกล:** ป้องกันการใส่ชื่อซ้ำในวันเดียวกัน และแจ้งเตือนกระพริบสีแดงหากมีการจัดเวรติดกัน 24 ชั่วโมง (ควงเวร) [cite: 756, 758, 775, 1054, 1057]

### 3. ระบบแลกเปลี่ยนเวร (Shift Swap)
* [cite_start]**การขอยกเวร:** เจ้าหน้าที่คลิกที่ป้ายเวรของตนเอง (แสดงผลเป็นสีทอง) เพื่อทำรายการขอยกเวรให้เพื่อนร่วมงานที่มีสิทธิ์ในหน้าที่เดียวกัน [cite: 853, 855, 1047]
* [cite_start]**การควบคุมเวลา:** ระบบจะซ่อนปุ่มเปลี่ยนเวร หากเป็นเวรที่ผ่านมาแล้วในอดีต (ซึ่ง Admin สามารถตั้งค่าเปิด-ปิดกฎข้อนี้ได้) [cite: 1044, 1046, 1059, 1063]
* [cite_start]**ประวัติไทม์ไลน์:** แสดงประวัติการแลกเปลี่ยนเวรเป็น Timeline พร้อมสถานะ (รออนุมัติ/อนุมัติแล้ว) เรียงจากใบล่าสุด [cite: 1030, 1034, 1035]
* [cite_start]**ระบบรันเลขเอกสาร:** สร้างเลขที่ใบเปลี่ยนเวรอัตโนมัติเมื่อทำรายการ (Running Number เช่น VC-2404-001) [cite: 1039, 1043]
* [cite_start]**การยกเลิก:** เจ้าหน้าที่สามารถกดยกเลิกใบคำขอที่ยังรออนุมัติ เพื่อดึงชื่อตนเองกลับเข้าเวรตามเดิมได้ [cite: 1099, 1100]

### 4. ระบบออกรายงานและเอกสาร (Export)
* [cite_start]**พิมพ์ตารางเวร:** แปลงหน้าปฏิทินสีสันสดใส ให้เป็นตารางตารางเอกสารขาว-ดำที่เป็นทางการ พร้อมช่องสำหรับให้ผู้บริหารลงนาม (Print Media Query) [cite: 845, 848]
* [cite_start]**ส่งออกใบเปลี่ยนเวร Word:** ดึงข้อมูลจากระบบไปเติมในไฟล์ Template (.docx) เพื่อให้เจ้าหน้าที่พรินต์ออกไปเซ็นชื่อตัวจริงได้ทันที [cite: 552, 553, 1099]

## 🚀 การติดตั้งและใช้งาน (Installation via Docker)

[cite_start]โปรเจกต์นี้ถูกออกแบบให้รันผ่าน Docker เพื่อความสะดวกในการจำลองสภาพแวดล้อมทั้ง Frontend, Backend และ Database [cite: 810, 815]

### การเริ่มต้นระบบ
1. [cite_start]ตรวจสอบให้แน่ใจว่าติดตั้ง Docker และ Docker Compose ในเครื่องแล้ว [cite: 810]
2. [cite_start]เปิด Terminal เข้าไปที่โฟลเดอร์นอกสุดของโปรเจกต์ (โฟลเดอร์ที่มีไฟล์ `docker-compose.yml`) [cite: 812]
3. รันคำสั่งต่อไปนี้เพื่อสร้างและเปิดระบบ:
   ```bash
   docker-compose up -d --build
   ```
   [cite_start][cite: 561, 817]

### การเข้าถึงระบบ
* [cite_start]**หน้าเว็บ (Frontend):** เข้าใช้งานระบบได้ที่ `http://localhost:5173` [cite: 560, 816]
* [cite_start]**Backend API:** ทำงานอยู่ที่ `http://localhost/vengg3/backend/public/index.php` [cite: 813, 816]
* [cite_start]**จัดการฐานข้อมูล:** เข้าถึง phpMyAdmin ได้ที่ `http://localhost:8080` (Username: `root` / Password: `root`) [cite: 560, 813, 819]

### การตั้งค่า Environment (เซิร์ฟเวอร์จริง)
[cite_start]หากต้องการนำระบบขึ้นเซิร์ฟเวอร์จริง (Production) ให้เข้าไปสร้างหรือแก้ไขไฟล์ `.env.production` ในโฟลเดอร์ `frontend` เพื่อระบุ IP หรือ Domain Name ของ Backend API แทนที่ `localhost` ระบบจะปรับใช้ค่านี้เมื่อทำการ Build [cite: 611, 613]
